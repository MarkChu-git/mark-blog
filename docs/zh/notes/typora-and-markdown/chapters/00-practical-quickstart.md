# 第零章：新手快学版——先把 Typora、Markdown、LaTeX 真正用起来



这不是“把整本教材压缩成一页”的偷懒版，也不是一张只有符号的速查表。

它的目标更明确：**让一个几乎没学过 Markdown 的新手，在最短时间里建立能马上用起来的写作工作流。**

如果你现在最想要的是：

- 赶紧开始写 Markdown，
- 先会最常用的语法，
- 先把 Typora 配顺手，
- 先知道 Git 日常怎么配合文档写作，
- 先会写一点最常见的 LaTeX 公式，

那么你应该先读这一章，再回去慢慢读完整教材。

---

## 一、这章适合什么人

这章特别适合下面几类读者：

- **完全新手**：第一次认真接触 Markdown / Typora。
- **想马上上手的人**：不想先读一大堆原理，想先把常用动作跑通。
- **文档型用户**：写笔记、教程、教材、博客、课程作业、实验报告的人。
- **实用主义读者**：优先关心“怎么马上写出来”，之后再补理论。

如果你已经知道 Markdown 的价值，但迟迟没真正用起来，通常不是因为你笨，而是因为你一开始接触到的内容顺序不对：**讲得太散、太抽象、太像语法表，而不像真正的工作流。**

这章就是为了纠正这个问题。

---

## 二、先记住最重要的结论

如果你今天只能带走 5 件事，那就是这 5 件：

1. **Markdown 最重要的不是花样，而是结构。**
2. **Typora 最有价值的地方，不是“好看”，而是让你边写源码边保持阅读感。**
3. **文档写作最常用的骨架只有几种：标题、段落、列表、引用、代码块、链接、图片、表格。**
4. **LaTeX 公式对新手来说，先会行内公式、块级公式、分式、上下标就够用了。**
5. **Git 不用一上来学复杂分支，先会 `status`、`add`、`commit`、`pull`、`push` 就能覆盖大量日常文档场景。**

先把这些基本动作练熟，你就已经超过了大量“看过教程但没形成习惯”的人。

---

## 三、30 分钟上手路线

建议你按这个顺序跑一遍：

### 第一步：装好 Typora，先不要折腾主题

先安装 Typora，然后做一件最重要的事：**新建一个文件，开始写。**

不要一上来折腾：

- 主题
- 字体
- 导出样式
- 插件感幻想象
- 花哨模板

真正决定你能不能坚持写下去的，不是这些，而是你能不能立刻形成“打开就写”的节奏。

### 第二步：学最常用的 8 类 Markdown 语法

你今天最该先会的是：

- 标题
- 段落
- 列表
- 引用
- 代码块
- 链接
- 图片
- 表格

### 第三步：把 LaTeX 只学到“够用”

先只会这些：

- 行内公式 `$...$`
- 块级公式 `$$...$$`
- 分式 `\frac{a}{b}`
- 上标 `x^2`
- 下标 `a_1`
- 根号 `\sqrt{x}`
- 求和 `\sum_{i=1}^{n}`

### 第四步：建立一个稳定的文档目录

至少这样组织：

```text
my-notes/
├── README.md
├── chapters/
│   └── 01-my-first-note.md
└── images/
    └── 01-example.png
```

### 第五步：用 Git 管住变化

哪怕你是一个人写文档，也建议你立刻开始用 Git，因为文档也会不断修改、删改、回滚、扩写。

## 四、先会写：最常用 Markdown 语法一把过

下面这些不是“所有语法”，而是**你最先必须会、而且最常用的语法**。

### 1. 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

什么时候用？

- `#`：一篇文档的标题
- `##`：一个主要部分
- `###`：一个部分里的小节

新手最常见错误：

- 不分层级，整篇全是二级标题
- 为了“看起来大”乱用一级标题
- 拿标题当加粗字用

你要记住：**标题不是字号，是结构。**

### 2. 段落

普通写作直接打字即可，换段时按回车开始新段。

```markdown
这是第一段。

这是第二段。
```

如果你只是想在同一段里强制换一行，通常可以用：

- `Shift + Enter`
- 或 `<br>`

### 3. 无序列表

```markdown
- 苹果
- 香蕉
- 橙子
```

适合表达并列项。

### 4. 有序列表

```markdown
1. 打开 Typora
2. 新建文档
3. 输入标题
4. 开始写正文
```

适合表达步骤。

### 5. 嵌套列表

```markdown
1. 安装环境
   - 安装 Typora
   - 安装 Git
2. 新建仓库
   - 建立 chapters 目录
   - 建立 images 目录
```

这里最重要的是缩进。**缩进代表层级，不是为了排版好看。**

### 6. 引用

```markdown
> 这是一段引用。
```

适合放：

- 提醒
- 定义
- 警告
- 摘录
- 旁注

例如：

```markdown
> 注意：图片尽量使用相对路径，不要用本机绝对路径。
```

### 7. 行内代码

```markdown
使用 `git status` 查看当前状态。
```

适合在句子中提命令、文件名、参数名。

### 8. 代码块

````markdown
```bash
git status
git add .
git commit -m "docs: update chapter"
```
````

你最常见会写的语言名：

- `bash`
- `python`
- `json`
- `yaml`
- `text`

### 9. 链接

```markdown
[Typora 官网](https://typora.io)
```

### 10. 图片

```markdown
![示意图](../images/example.png)
```

这里最重要的不是“会不会插图”，而是：

- 图片有没有统一放在 `images/`
- 路径是不是相对路径
- 文件名是不是有规律

### 11. 表格

```markdown
| 语法 | 作用 |
| --- | --- |
| # 标题 | 定义标题层级 |
| - 列表 | 表示无序项 |
```

表格适合做：

- 对照
- 总结
- 速览

但不适合塞太长的段落。

### 12. 强调

```markdown
**粗体**
*斜体*
~~删除线~~
```

强调是点缀，不是骨架。

## 五、直接可抄的 Markdown 写作模板

如果你现在就要开始写一篇教程，最简单的模板可以先用这个：

```markdown
# 我的第一篇 Markdown 文档

## 1. 这篇文档要解决什么问题

先用一两段说明这篇文档的目标。

## 2. 需要准备什么

- 工具 A
- 工具 B
- 一个示例文件

## 3. 实际步骤

### 3.1 第一步

写清楚动作。

### 3.2 第二步

继续写动作。

## 4. 常见问题

> 注意：这里放最容易踩坑的地方。

## 5. 小结

最后总结你真正希望读者记住的东西。
```

这个模板不华丽，但非常稳。

## 六、LaTeX 公式：新手只学最有用的

很多人一看到公式就怂，其实大部分文档里真正常用的公式写法并不多。

## 1. 行内公式

```markdown
$f = \frac{1}{T}$
```

效果：把公式嵌在句子里。

例如：频率可以写成 $f = \frac{1}{T}$。

## 2. 块级公式

```markdown
$$
E = mc^2
$$
```

适合单独展示公式。

## 3. 分式

```markdown
$\frac{a+b}{c}$
```

## 4. 上标与下标

```markdown
$x^2$
$a_1$
```

## 5. 根号

```markdown
$\sqrt{x^2+y^2}$
```

## 6. 求和

```markdown
$\sum_{i=1}^{n} i$
```

## 7. 积分

```markdown
$\int_a^b f(x)\,dx$
```

## 8. 多行对齐

```markdown
$$
\begin{align}
a+b &= c \\
d &= e+f
\end{align}
$$
```

新手阶段真的先会这些就够用了。

## 七、Typora 里最值得先开的设置

你不需要把所有设置研究一遍，但下面这些很值得先看：

### 1. Markdown 相关设置

重点看：

- 是否开启行内公式（inline math）
- 是否显示大纲（outline）
- 是否启用文件树（file tree）
- 图片插入时如何处理路径

### 2. 图片设置

如果你准备长期写文档，建议优先形成这种习惯：

- 图片复制到固定目录
- 路径用相对路径
- 图片文件名有规则

例如：

- `chapter-01-figure-01.png`
- `chapter-01-figure-02.png`

### 3. 学几个高频快捷键

先会最常用的，不要追求一口气全背：

- 新建标题
- 插入列表
- 插入代码块
- 插入表格
- 搜索
- 打开大纲

## 八、文档工作流：不是写完就完，而是持续维护

真正的文档工作流通常不是“写一篇就结束”，而是：

- 先写初稿
- 再补结构
- 再补图
- 再修语言
- 再调整目录
- 之后继续增补

这就是为什么 Markdown + Typora 适合长期维护。

你写的是纯文本，结构清晰，适合放进 Git，也适合日后拆分、重组、导出、协作。

---

## 九、Git 先只学最常用一套

如果你今天是第一次碰 Git，不要一上来学复杂分支模型。

先会这套最小工作流：

### 1. 初始化仓库

```bash
git init
```

### 2. 看状态

```bash
git status
```

### 3. 添加改动

```bash
git add .
```

### 4. 提交

```bash
git commit -m "docs: add first markdown draft"
```

### 5. 连接远程仓库（如果还没配）

```bash
git remote add origin <your-repo-url>
```

### 6. 推送

```bash
git push -u origin main
```

### 7. 拉取

```bash
git pull
```

这 7 个动作已经能覆盖大量个人文档维护场景。

## 十、文档作者最常用的 Git 命令

下面这张表建议你反复看，直到形成肌肉记忆。

| 命令 | 作用 | 最常见场景 |
| --- | --- | --- |
| `git status` | 看当前改动状态 | 每次提交前先看一眼 |
| `git add .` | 把改动加入暂存区 | 准备提交 |
| `git commit -m "..."` | 记录一次变化 | 每完成一个小阶段就提交 |
| `git pull` | 拉远程最新改动 | 开工前先同步 |
| `git push` | 把本地提交推到远程 | 提交完成后同步 |
| `git log --oneline` | 看提交历史 | 回看最近做了什么 |
| `git diff` | 看改了什么 | 提交前检查具体差异 |

## 十一、一个最实用的新手日常节奏

如果你是写文档的人，日常最推荐这样干：

### 开工前

```bash
git pull
```

### 写作中

- 写完一小段结构
- 写完一个小节
- 改完一批图
- 改完一轮语言

就做一次提交。

### 提交时

```bash
git add .
git commit -m "docs: refine chapter 1 structure"
```

### 收工前

```bash
git push
```

这样你会很快进入一种稳定状态：**每次修改都有记录，每次回退都有抓手。**

## 十三、本章小结

这一章你最该记住的是：

- 先学会最常用的动作，不要先追求全会。
- Markdown 的关键是结构，不是炫技。
- LaTeX 先学最常见的几种表达就够了。
- Typora 先配最影响写作体验的设置。
- Git 先会最小闭环：`status` → `add` → `commit` → `pull` / `push`。

你只要把这套最小工作流跑顺，后面再读完整教材时，理解速度会快很多，而且会更有兴趣，因为你已经真的用起来了。
