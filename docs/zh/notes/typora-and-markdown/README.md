# 《Typora 与 Markdown》

本册的目标不是把 Markdown 写成一张语法小抄，也不是把 Typora 写成一篇软件推荐帖，而是系统讲清楚这些问题：为什么要这样写文档，为什么很多人学了语法却仍然写不好，Typora 在这条路上究竟解决了什么问题，以及如何把 Markdown、图片、目录、LaTeX、Git 工作流接成一个长期可维护的文档系统。

## 适合谁读

- 从没认真学过 Markdown，但已经意识到自己需要更稳定写作方式的人
- 已经会一点 Markdown 语法，却仍然经常在排版、路径、导出和平台兼容性上踩坑的人
- 想把 Typora 作为长期知识整理工具来用，而不是只把它当作一个"能写 Markdown 的编辑器"的人

## 快学实战线

本册提供了一条**快学实战线**：先给出一份面向新手的快学章节，强调最常用语法、LaTeX 基础、图片路径、Typora 设置、Git 配合文档写作，让读者先能写、先能用，再回头读完整教材。

## 写作范围

1. Markdown 的设计思想与写作观
2. Typora 的基本界面、输入逻辑与写作体验
3. Markdown、CommonMark、GitHub Flavored Markdown 与 Typora 支持集之间的关系
4. 标题、段落、列表、引用、代码块、链接、图片、表格、脚注、公式等核心写法
5. 图片路径、相对路径、目录结构、文件命名与文档组织
6. YAML Front Matter、导出、PDF/HTML、GitHub 展示差异
7. 从单篇文档写作走向长期教材和知识库维护的方法
8. 面向新手的快学实战工作流
